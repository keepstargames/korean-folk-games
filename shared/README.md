# shared/ — 게임 공통 모듈 자리

여러 게임이 공유하게 될 코드를 두는 폴더입니다.
**현재는 비어 있습니다.** 아직 어떤 게임에서도 코드를 빼내지 않았습니다.

## 왜 지금은 비어 있나

추출은 게임이 **두 개 이상** 됐을 때 합니다.
게임 하나만 있는 상태에서 미리 공통 모듈을 뽑으면, 두 번째 게임이 실제로
무엇을 필요로 하는지 모른 채 구조를 정하게 되어 십중팔구 다시 뜯어고치게 됩니다.
두 번째 게임(고누 예정)을 만들면서 **실제로 중복되는 것**이 드러나면 그때 옮깁니다.

`games/yutnori/index.html`은 지금처럼 단일 파일로 자체 완결 상태를 유지합니다.

## 추출 후보 (2번째 게임 때 판단)

CLAUDE.md의 공통 규칙에서 이미 모든 게임이 공유하기로 정해진 것들:

| 후보 | 지금 위치 | 비고 |
|---|---|---|
| `i18n.js` | yutnori의 `L{}` 딕셔너리 + `t()` + `applyLang()` | 한/영 전환 방식이 모든 게임 공통 |
| `tokens.css` | yutnori의 `:root` 변수 9개 | 한지·먹 디자인 토큰. index.html에도 이미 중복됨 |
| `sfx.js` | yutnori의 WebAudio 합성 코드 | 외부 오디오 파일 금지 규칙상 모든 게임이 합성 필요 |
| `hanji.css` | 한지 질감 배경, 낙관(seal), 폰트 스택 | |

> 실제로 중복이 확인되기 전까지 이 표는 **예상**일 뿐입니다. 확정 아님.

## 지킬 것 (CLAUDE.md 규칙)

- 외부 오디오/이미지 파일 금지 — 사운드는 WebAudio 합성, 그래픽은 SVG/CSS
- `localStorage` / `sessionStorage` 금지 — 세션 내 메모리만
- 빌드 도구 없이 브라우저에서 바로 동작해야 함 (정적 호스팅 대상)

마지막 항목 때문에 추출 시 **ES 모듈**(`<script type="module">`)이나 평범한
`<script>` 전역 방식을 쓰게 됩니다. 번들러는 도입하지 않습니다.
